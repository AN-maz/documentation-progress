## 1. Scenario: Membayar Tagihan Internet

| Atribut                      | Keterangan                                                                                                                                                                                                                                                                                                                                                            |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case Name**            | Membayar Tagihan Internet                                                                                                                                                                                                                                                                                                                                             |
| **ID**                       | UC_SA01                                                                                                                                                                                                                                                                                                                                                               |
| **Importance Level**         | High                                                                                                                                                                                                                                                                                                                                                                  |
| **Primary Actor**            | Pelanggan                                                                                                                                                                                                                                                                                                                                                             |
| **Secondary Actor**          | -                                                                                                                                                                                                                                                                                                                                                                     |
| **Use Case Type**            | Main Case                                                                                                                                                                                                                                                                                                                                                             |
| **Stakeholder and Interest** | **Pelanggan:** Ingin melunasi tagihan internet bulanannya dengan cepat menggunakan saldo C-Pay.                                                                                                                                                                                                                                                                       |
| **Brief Description**        | Di dalam _use case_ ini dijelaskan alur pelanggan memeriksa tagihan internet yang aktif dan melakukan pembayaran menggunakan saldo C-Pay.                                                                                                                                                                                                                             |
| **Trigger**                  | Pelanggan membuka menu pembayaran tagihan pada aplikasi.                                                                                                                                                                                                                                                                                                              |
| **Type**                     | External                                                                                                                                                                                                                                                                                                                                                              |
| **Relationship**             | **Association:** Pelanggan<br>**Include:** -<br>**Extend:** Melakukan Top Up Saldo C-Pay (UC_SA02)                                                                                                                                                                                                                                                                    |
| **Normal Flow of Events**    | 1. Pelanggan memilih menu **"Bayar Internet"** pada aplikasi.<br>2. Sistem menampilkan informasi tagihan yang belum dibayarkan.<br>3. Pelanggan memilih metode pembayaran menggunakan C-Pay.<br>4. Sistem memverifikasi ketersediaan saldo C-Pay.<br>5. Sistem memproses pembayaran, memperbarui status tagihan menjadi **"Lunas"**, dan menampilkan bukti transaksi. |
| **Alternate Flows**          | 4A. Saldo C-Pay tidak mencukupi. Sistem memicu alur perluasan (_Extend_) menuju **Melakukan Top Up Saldo C-Pay (UC_SA02)**.                                                                                                                                                                                                                                           |
| **Exceptional Flows**        | 2E. Data pelanggan atau tagihan tidak ditemukan. Sistem menampilkan pesan kesalahan dan membatalkan proses pembayaran.                                                                                                                                                                                                                                                |

## 3. Scenario: Memvalidasi Transaksi Masuk

| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case Name**            | Memvalidasi Transaksi Masuk                                                                                                                                                                                                                                                                                       |
| **ID**                       | UC_SA03                                                                                                                                                                                                                                                                                                           |
| **Importance Level**         | High                                                                                                                                                                                                                                                                                                              |
| **Primary Actor**            | Finance                                                                                                                                                                                                                                                                                                           |
| **Secondary Actor**          | -                                                                                                                                                                                                                                                                                                                 |
| **Use Case Type**            | Main Case                                                                                                                                                                                                                                                                                                         |
| **Stakeholder and Interest** | **Finance:** Ingin memastikan seluruh transaksi yang diterima sistem sesuai dengan data mutasi keuangan perusahaan.                                                                                                                                                                                               |
| **Brief Description**        | Di dalam _use case_ ini dijelaskan alur staf Finance dalam memeriksa, memvalidasi, dan mengesahkan transaksi yang masuk ke sistem.                                                                                                                                                                                |
| **Trigger**                  | Finance melakukan pemeriksaan laporan transaksi harian.                                                                                                                                                                                                                                                           |
| **Type**                     | Internal                                                                                                                                                                                                                                                                                                          |
| **Relationship**             | **Association:** Finance<br>**Include:** -<br>**Extend:** -                                                                                                                                                                                                                                                       |
| **Normal Flow of Events**    | 1. Finance membuka dashboard transaksi.<br>2. Sistem menampilkan daftar transaksi yang telah diterima.<br>3. Finance mencocokkan data transaksi dengan laporan keuangan yang tersedia.<br>4. Finance menyetujui hasil validasi transaksi.<br>5. Sistem menyimpan hasil validasi dan memperbarui laporan keuangan. |
| **Alternate Flows**          | 3A. Terdapat transaksi yang belum terverifikasi secara otomatis. Finance melakukan validasi secara manual sebelum menyetujui transaksi tersebut.                                                                                                                                                                  |
| **Exceptional Flows**        | 3E. Ditemukan ketidaksesuaian data transaksi yang signifikan. Sistem menandai transaksi sebagai bermasalah dan menghentikan proses validasi hingga dilakukan pemeriksaan lebih lanjut.                                                                                                                            |
