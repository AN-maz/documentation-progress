# Sistem Visit Track

## 1. Scenario: Mencatat Koordinat Lokasi Kunjungan

| Atribut                      | Keterangan                                                                                                                                                                                                                                                                                                         |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Use Case Name**            | Mencatat Koordinat Lokasi Kunjungan                                                                                                                                                                                                                                                                                |
| **ID**                       | UC_VT01                                                                                                                                                                                                                                                                                                            |
| **Importance Level**         | High                                                                                                                                                                                                                                                                                                               |
| **Primary Actor**            | Sales                                                                                                                                                                                                                                                                                                              |
| **Secondary Actor**          | -                                                                                                                                                                                                                                                                                                                  |
| **Use Case Type**            | Main Case                                                                                                                                                                                                                                                                                                          |
| **Stakeholder and Interest** | **Sales:** Ingin mencatat lokasi kunjungannya sebagai bukti kerja lapangan dengan akurat.<br>**Manajemen:** Ingin melacak aktivitas dan keabsahan titik lokasi kunjungan Sales.                                                                                                                                    |
| **Brief Description**        | Di dalam *use case* ini dijelaskan alur Sales merekam titik koordinat GPS lokasi kunjungan calon pelanggan menggunakan aplikasi VisitTrack.                                                                                                                                                                        |
| **Trigger**                  | Sales tiba di lokasi calon pelanggan.                                                                                                                                                                                                                                                                              |
| **Type**                     | External                                                                                                                                                                                                                                                                                                           |
| **Relationship**             | **Association:** Sales<br>**Include:** -<br>**Extend:** -                                                                                                                                                                                                                                                          |
| **Normal Flow of Events**    | 1. Sales membuka aplikasi VisitTrack dan memilih menu **Catat Kunjungan**.<br>2. Sistem mendeteksi lokasi GPS Sales saat ini.<br>3. Sistem menampilkan titik koordinat dan alamat pada peta.<br>4. Sales menekan tombol **Simpan Koordinat**.<br>5. Sistem menyimpan data koordinat kunjungan ke dalam basis data. |
| **Alternate Flows**          | 2A. Sistem tidak dapat mendeteksi lokasi secara otomatis.<br>2B. Sales menentukan titik lokasi secara manual pada peta.                                                                                                                                                                                            |
| **Exceptional Flows**        | 2E. Koneksi internet atau sensor GPS pada perangkat Sales tidak aktif. Sistem menampilkan pesan kesalahan dan membatalkan proses pencatatan lokasi.                                                                                                                                                                |

## 2. Scenario: Mencatat Data Prospek Pelanggan (Status dan Keluhan)

| Atribut                      | Keterangan                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case Name**            | Mencatat Data Prospek Pelanggan (Status dan Keluhan)                                                                                                                                                                                                                                                                                                     |
| **ID**                       | UC_VT02                                                                                                                                                                                                                                                                                                                                                  |
| **Importance Level**         | High                                                                                                                                                                                                                                                                                                                                                     |
| **Primary Actor**            | Sales                                                                                                                                                                                                                                                                                                                                                    |
| **Secondary Actor**          | Calon Pelanggan                                                                                                                                                                                                                                                                                                                                          |
| **Use Case Type**            | Main Case                                                                                                                                                                                                                                                                                                                                                |
| **Stakeholder and Interest** | **Sales:** Ingin mendata status minat dan keluhan calon pelanggan ke dalam sistem.<br>**Calon Pelanggan:** Ingin keluhan dan kondisi jaringan yang dialaminya tersampaikan kepada penyedia layanan.                                                                                                                                                      |
| **Brief Description**        | Di dalam *use case* ini dijelaskan alur Sales melakukan wawancara dengan calon pelanggan dan memasukkan data status prospek serta keluhan ke dalam sistem.                                                                                                                                                                                               |
| **Trigger**                  | Sales selesai melakukan wawancara dengan calon pelanggan.                                                                                                                                                                                                                                                                                                |
| **Type**                     | External                                                                                                                                                                                                                                                                                                                                                 |
| **Relationship**             | **Association:** Sales, Calon Pelanggan<br>**Include:** -<br>**Extend:** Mengajukan Eskalasi Pemasangan (UC_VT03)                                                                                                                                                                                                                                        |
| **Normal Flow of Events**    | 1. Sales mewawancarai calon pelanggan.<br>2. Calon pelanggan menyampaikan informasi mengenai kondisi internet dan keluhan yang dialami.<br>3. Sales membuka formulir pendataan prospek pada aplikasi.<br>4. Sales memasukkan catatan keluhan dan memilih kategori status prospek.<br>5. Sistem memproses dan menyimpan data prospek ke dalam basis data. |
| **Alternate Flows**          | 4A. Sales memilih kategori status **Hot**, sehingga sistem memicu *use case* **Mengajukan Eskalasi Pemasangan (UC_VT03)**.                                                                                                                                                                                                                               |
| **Exceptional Flows**        | 4E. Data prospek tidak diisi secara lengkap. Sistem menolak penyimpanan dan meminta Sales melengkapi data yang diperlukan.                                                                                                                                                                                                                               |

## 3. Scenario: Mengajukan Eskalasi Pemasangan

| Atribut                      | Keterangan                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Use Case Name**            | Mengajukan Eskalasi Pemasangan                                                                                                                                                                                                                                                                                                                                                            |
| **ID**                       | UC_VT03                                                                                                                                                                                                                                                                                                                                                                                   |
| **Importance Level**         | High                                                                                                                                                                                                                                                                                                                                                                                      |
| **Primary Actor**            | Sales                                                                                                                                                                                                                                                                                                                                                                                     |
| **Secondary Actor**          | -                                                                                                                                                                                                                                                                                                                                                                                         |
| **Use Case Type**            | Extension Case                                                                                                                                                                                                                                                                                                                                                                            |
| **Stakeholder and Interest** | **Sales:** Ingin meneruskan data calon pelanggan yang sangat berminat agar segera diproses oleh tim teknis.<br>**Tim Operasional:** Ingin menerima data calon pelanggan dengan informasi yang lengkap dan valid.                                                                                                                                                                          |
| **Brief Description**        | Di dalam *use case* ini dijelaskan alur pengajuan eskalasi pemasangan ketika calon pelanggan memiliki minat tinggi untuk berlangganan layanan internet.                                                                                                                                                                                                                                   |
| **Trigger**                  | Sales memilih kategori status prospek **Hot** pada proses pendataan pelanggan.                                                                                                                                                                                                                                                                                                            |
| **Type**                     | Internal                                                                                                                                                                                                                                                                                                                                                                                  |
| **Relationship**             | **Association:** Sales<br>**Include:** -<br>**Extend:** Perluasan (*Extend*) dari **Mencatat Data Prospek Pelanggan (UC_VT02)**                                                                                                                                                                                                                                                           |
| **Normal Flow of Events**    | 1. Sistem menampilkan formulir eskalasi pemasangan.<br>2. Sales melengkapi data pendukung pelanggan.<br>3. Sales menekan tombol **Eskalasi ke Tim Teknis**.<br>4. Sistem memverifikasi kelengkapan data.<br>5. Sistem membuat tiket pemasangan berdasarkan data pelanggan dan lokasi kunjungan.<br>6. Sistem mengirimkan tiket ke sistem operasional dan menampilkan notifikasi berhasil. |
| **Alternate Flows**          | -                                                                                                                                                                                                                                                                                                                                                                                         |
| **Exceptional Flows**        | 4E. Data pendukung yang wajib diisi belum lengkap. Sistem menampilkan peringatan dan menunda proses pengiriman tiket hingga data dilengkapi.                                                                                                                                                                                                                                              |

Versi ini sudah konsisten dengan dokumen sebelumnya:

* Semua komponen berada dalam satu tabel

* Menggunakan istilah **basis data** alih-alih campuran *database*
* Format **Normal / Alternate / Exceptional Flows** sama seperti UC_ABS, UC_SA, dan UC_OP yang sudah dirapikan sebelumnya.
