# Type Casting 
Dalam arsitektur bahasa pemrograman yang diketik secara dinamis (dynamically typed) seperti JavaScript, nilai dari sebuah variabel dapat mengalami transformasi dari satu tipe data ke tipe data lain. Mekanisme transformasi ini secara luas dikenal sebagai Type Casting.

Untuk memahami bagaimana JavaScript menangani interaksi antar tipe data yang berbeda, kita harus membedah konsep ini ke dalam dikotomi fundamental: Type Conversion dan Type Coercion.

## 1. Type Conversion vs. Type Coercion

Meskipun secara terminologis sering digunakan secara bergantian dalam literatur non-formal, dalam spesifikasi ECMAScript, keduanya merujuk pada dua mekanisme komputasi yang berbeda berdasarkan intervensi entitas yang melakukannya:

- Type Conversion (Konversi Eksplisit): Proses transformasi tipe data yang diinstruksikan secara sengaja dan eksplisit oleh programmer melalui penulisan sintaks atau pemanggilan fungsi bawaan (built-in functions).

- Type Coercion (Koersi Implisit): Proses transformasi tipe data yang dieksekusi secara otomatis (under the hood) oleh JavaScript engine (seperti V8 Engine) pada saat runtime. Hal ini terjadi ketika sebuah operator digunakan pada operand (nilai yang dioperasikan) yang memiliki tipe data heterogen (berbeda), sehingga engine harus "memaksa" (coerce) salah satu nilai agar operasi tersebut valid secara matematis atau logis.   


## 2. Implicit Type Casting (Type Coercion)
Koersi implisit sering kali menjadi sumber anomali atau bug jika programmer tidak memahami regulasi internal JavaScript engine. Koersi umumnya terbagi menjadi tiga kategori operasional: koersi ke String, ke Number, dan ke Boolean.

### a. String Coercion (Melalui Operator Penambahan +)
Ketika operator `+` mempertemukan tipe data `String` dengan tipe data lain (seperti``Number`), engine akan memprioritaskan penyatuan string (string concatenation). Nilai numerik akan dikoersi menjadi representasi tekstualnya.

    const angka = 50;
    const teks = " adalah angka";

    // JavaScript secara implisit mengubah '50' (Number) menjadi "50" (String)
    const hasil = angka + teks; 
    console.log(hasil); // Output: "50 adalah angka" (Tipe: String)

### b. Number Coercion (Melalui Operator Aritmatika Selain `+`)

Ketika operator aritmatika seperti `pengurangan (-)`, `perkalian (*)`, atau `pembagian (/)` digunakan, JavaScript mengasumsikan niat matematis murni. Oleh karena itu, engine akan mengoersi string yang berisi karakter numerik menjadi tipe Number.

    const nilaiString = "100";
    const nilaiPengurang = 25;

    // JavaScript secara implisit mengubah "100" (String) menjadi 100 (Number)
    const kalkulasi = nilaiString - nilaiPengurang;
    console.log(kalkulasi); // Output: 75 (Tipe: Number)

    // Jika string tidak merepresentasikan angka valid, akan menghasilkan NaN (Not a Number)
    const gagalKalkulasi = "Teks" * 5;
    console.log(gagalKalkulasi); // Output: NaN

### C. Boolean Coercion (Evaluasi Truthy dan Falsy)
Dalam konteks percabangan logika (seperti if statement), JavaScript akan mengevaluasi setiap tipe data ke dalam nilai boolean absolut (true atau false).

- `Falsy values`: Terdapat himpunan nilai spesifik yang secara inheren akan dievaluasi sebagai `false`: `0`, `""` (string kosong), `null`, `undefined`, `NaN`, dan `false` itu sendiri.

- `Truthy values`: Semua entitas data di luar himpunan falsy secara mutlak akan dievaluasi sebagai `true` (termasuk array kosong `[]` dan objek kosong `{}`).


        let dataTidakTerdefinisi; // Nilai default: undefined

        if (dataTidakTerdefinisi) {
            // Blok ini diabaikan karena 'undefined' dikoersi menjadi false (Falsy)
            console.log("Data tersedia.");
        } else {
            console.log("Data kosong atau falsy."); // Output ini yang tereksekusi
        }

## 3. Explicit Type Casting (Type Conversion)

Untuk menjaga integritas dan prediktabilitas kode, best practice dalam rekayasa perangkat lunak menganjurkan penggunaan konversi eksplisit. JavaScript menyediakan fungsi wrapper (pembungkus) global untuk mengeksekusi konversi ini secara deterministik.

### a. Konversi ke String
Menggunakan fungsi `String()` atau method `.toString()`.

    const numerik = 2026;

    // Konversi eksplisit
    const representasiString1 = String(numerik);
    const representasiString2 = numerik.toString();

    console.log(typeof representasiString1); // Output: "string"

### b. Konversi ke Number

Menggunakan fungsi `Number()`, atau fungsi parsing yang lebih spesifik seperti `parseInt()` (untuk mengambil bilangan bulat) dan `parseFloat()` (untuk mempertahankan bilangan desimal/pecahan).

    const inputPengguna = "3.14";

    const konversiUtuh = Number(inputPengguna);      // Output: 3.14
    const konversiBulat = parseInt(inputPengguna);   // Output: 3 (Desimal diabaikan)
    const konversiDesimal = parseFloat(inputPengguna); // Output: 3.14

    // Mengubah tipe boolean ke number
    const booleanKeNumerik = Number(true); // Output: 1 (false akan menjadi 0)

### c. Konversi ke Boolean

Menggunakan fungsi `Boolean()`. Ini pada dasarnya menjalankan mekanisme evaluasi truthy dan falsy secara manual.

    const validasi1 = Boolean(100);    // Output: true (Truthy)
    const validasi2 = Boolean("");     // Output: false (Falsy)
    const validasi3 = Boolean("0");    // Output: true (String tidak kosong adalah Truthy)
