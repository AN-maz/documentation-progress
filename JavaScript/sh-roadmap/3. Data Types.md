# Data Types
Dalam literatur dan dokumentasi teknis ilmu komputer, JavaScript diklasifikasikan sebagai bahasa pemrograman yang diketik secara dinamis (dynamically typed language). Artinya, tipe data tidak diasosiasikan dengan variabel saat deklarasi, melainkan dievaluasi pada saat eksekusi (runtime).

Karakteristik tipe data dalam JavaScript secara fundamental direpresentasikan ke dalam dua kategori utama: Tipe Data Primitif (Primitive Types) dan Tipe Data Objek (Reference Types).

Berikut adalah elaborasi komprehensif mengenai struktur dan hierarki tipe data tersebut.

## 1. Tipe Data Primitif 

Tipe data primitif adalah nilai dasar yang berada di tingkat paling rendah dalam implementasi bahasa. Karakteristik utamanya adalah bersifat mutlak dan immutable (nilainya tidak dapat dimodifikasi secara langsung di dalam memori setelah dialokasikan). JavaScript mengimplementasikan tujuh tipe data primitif:

- `String`: Representasi data tekstual. Secara arsitektural, string di JavaScript adalah urutan nilai elemen 16-bit yang mematuhi standar pengkodean karakter UTF-16.

- `Number`: JavaScript tidak membedakan antara bilangan bulat (integer) dan bilangan pecahan (floating-point). Seluruh nilai numerik direpresentasikan menggunakan standar format double-precision 64-bit floating-point (IEEE 754).

- `BigInt`: Entitas primitif yang dikembangkan untuk merepresentasikan dan memanipulasi bilangan bulat komputasi yang melampaui batas representasi aman dari tipe Number (lebih besar dari $2^{53} - 1$).

- `Boolean`: Entitas logika biner yang hanya memiliki dua kemungkinan status komputasi: true (benar) atau false (salah).

- `Undefined`: Tipe data yang secara otomatis dialokasikan oleh mesin eksekusi (JavaScript Engine) pada sebuah variabel yang telah diinisialisasi melalui deklarasi memori, namun belum diikat (bound) dengan nilai apa pun.

- `Null`: Representasi intensional (intentional absence) dari ketiadaan nilai objek. Penggunaan null mengindikasikan bahwa variabel tersebut secara eksplisit diarahkan pada ketiadaan referensi.

- `Symbol`: Diperkenalkan pada ECMAScript 6 (ES6), tipe ini merepresentasikan pengenal (identifier) yang terjamin keunikannya. Umumnya dieksploitasi untuk menghindari kolisi penamaan pada kunci properti (property keys) dari sebuah objek.


## 2. Objek (Reference Types)

Berbeda dengan tipe data primitif yang berorientasi pada nilai (value), objek adalah entitas referensi komposit yang difungsikan untuk menyimpan koleksi data dan entitas yang lebih kompleks. Objek secara struktural adalah pemetaan antara kunci (key) dan nilai (value).

- Konsep Objek dan Object Prototype:

JavaScript mengimplementasikan paradigma berorientasi objek melalui mekanisme yang disebut Prototypal Inheritance (Pewarisan Prototipe). Setiap instansiasi objek di JavaScript memiliki tautan internal tersembunyi ([[Prototype]]) yang merujuk pada objek lain, yang disebut sebagai prototype.

- Prototypal Inheritance:

Ketika sebuah operasi mencoba mengakses properti pada suatu objek dan properti tersebut tidak terdefinisi pada ruang lingkup objek itu sendiri, kompilator (interpreter) akan secara otomatis mencari properti tersebut ke atas melalui rantai prototipe (prototype chain) hingga menemukan properti yang dimaksud, atau proses pencarian mencapai objek yang prototipenya bernilai `null` (mengembalikan `undefined`).

- Built-in Object (Objek Bawaan):

Spesifikasi ECMAScript menyediakan berbagai objek fundamental yang terintegrasi secara native pada lingkungan eksekusi untuk memfasilitasi operasi komputasi. Contohnya meliputi `Array` (untuk struktur data terindeks), `Date` (untuk manipulasi waktu temporal), `Math` (kumpulan fungsi matematika standar), serta struktur data lanjutan seperti Map dan Set.

## 3. Operator typeof

Operator typeof adalah operator unary bawaan yang mengevaluasi struktur operand dan mengembalikan representasi literal string yang mengindikasikan tipe data dari operand tersebut sebelum runtime.

Operator ini sangat krusial dalam paradigma dynamic typing untuk melakukan validasi input (inspeksi tipe).

    typeof "Algoritma"; // Mengembalikan: "string"
    typeof 404;         // Mengembalikan: "number"
    typeof true;        // Mengembalikan: "boolean"
    typeof undefined;   // Mengembalikan: "undefined"
    typeof Symbol("id"); // Mengembalikan: "symbol"

### Anomali Historis pada typeof null
Apabila Anda mengevaluasi typeof null, mesin JavaScript akan mengembalikan string "object". Dalam literatur arsitektur perangkat lunak JavaScript, perilaku ini diakui secara universal sebagai cacat desain (design flaw) atau bug dari implementasi awal bahasa ini. Namun, hal ini tidak diperbaiki secara permanen pada rilis modern demi menjaga kompatibilitas ke belakang (backward compatibility) agar sistem-sistem lama yang bergantung pada perilaku komputasi ini tidak mengalami malfungsi.
