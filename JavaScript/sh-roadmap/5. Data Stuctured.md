# Data Structures

Dalam ilmu komputer, struktur data adalah metode pengorganisasian, manajemen, dan penyimpanan data untuk memfasilitasi akses dan modifikasi yang efisien.

JavaScript menyediakan berbagai struktur data bawaan yang dirancang untuk mengoptimalkan skenario komputasi tertentu. Berikut adalah tinjauan akademis beserta implementasi sintaksisnya.

## 1. Structured Data: JSON (javaScript Notation)
JSON bukanlah struktur data di dalam memori eksekusi JavaScript itu sendiri, melainkan sebuah format standar berbasis teks untuk representasi data terstruktur (structured data serialization). JSON diturunkan dari sintaks objek literal JavaScript, namun bersifat independen dari bahasa (dapat dibaca oleh Python, Java, dll).

Dalam arsitektur client-server, JSON adalah instrumen utama untuk pertukaran data (API). Mesin JavaScript menyediakan objek global JSON yang memiliki dua metode statis utama untuk transformasi data:

- Serialisasi (`JSON.stringify`): Mengubah objek di dalam memori (in-memory object) menjadi untaian teks JSON untuk ditransmisikan.

- Deserialisasi (`JSON.parse`): Mengonversi untaian teks JSON yang diterima kembali menjadi objek komputasi JavaScript.

contoh

    // Representasi Objek JavaScript di memori
    const konfigurasi = {
        tema: "gelap",
        versi: 2.1,
        aktif: true
    };

    // 1. Serialisasi (Mengubah ke String JSON)
    const jsonString = JSON.stringify(konfigurasi);
    console.log(jsonString); 
    // Output: '{"tema":"gelap","versi":2.1,"aktif":true}' (Tipe: String)

    // 2. Deserialisasi (Mengubah String JSON kembali ke Objek)
    const objekParsed = JSON.parse(jsonString);
    console.log(objekParsed.tema); // Output: "gelap" (Tipe: Object)

## 2. Keyed Collections (Koleksi Berbasis Kunci)

Kategori ini merepresentasikan struktur data yang mengorganisir elemen tidak berdasarkan urutan indeks, melainkan berdasarkan pemetaan referensi (key-value) atau keunikan elemen.

### a. Map
Berbeda dengan Objek standar di mana kunci (key) dikoersi menjadi String atau Symbol, Map adalah struktur data dictionary sejati. Map mengizinkan tipe data apa pun (termasuk fungsi atau objek lain) untuk bertindak sebagai kunci. Map juga memelihara urutan insersi (urutan saat data dimasukkan).

    const memoriMap = new Map();
    const referensiObjek = { id: 1 };

    // Memasukkan data (set)
    memoriMap.set("stringKey", "Nilai 1");
    memoriMap.set(referensiObjek, "Data rahasia terkait objek ini"); // Kunci berupa objek!

    // Mengambil data (get)
    console.log(memoriMap.get(referensiObjek)); // Output: "Data rahasia terkait objek ini"
    console.log(memoriMap.size); // Output: 2

### b. WeakMap
`WeakMap` memiliki arsitektur yang mirip dengan `Map`, namun dengan restriksi ketat: kunci harus berupa objek dan tidak bisa berupa nilai primitif. Secara fungsional, `WeakMap` memegang referensi "lemah" (weak reference) terhadap objek kuncinya.
Ini berarti jika tidak ada lagi referensi lain di dalam program yang menunjuk ke objek kunci tersebut, mekanisme manajemen memori (Garbage Collector) akan secara otomatis menghapus entri tersebut dari `WeakMap` untuk mencegah kebocoran memori (memory leak). Karena sifat dinamis ini, WeakMap tidak dapat diiterasi (tidak punya metode perulangan seperti `forEach`).

    const cacheMemori = new WeakMap();
    let sesiPengguna = { user: "Admin" };

    cacheMemori.set(sesiPengguna, "Data Cache Sesi Aktif");

    // Jika sesi dihancurkan:
    sesiPengguna = null; 
    // Entri di dalam cacheMemori akan otomatis dibersihkan oleh Garbage Collector

### C. set
`Set` adalah koleksi matematika yang hanya menyimpan nilai-nilai unik dari tipe data apa pun. Jika Anda mencoba memasukkan nilai duplikat yang sudah eksis di dalam `Set`, kompilator akan mengabaikannya secara diam-diam.

    const koleksiUnik = new Set();

    koleksiUnik.add(10);
    koleksiUnik.add(20);
    koleksiUnik.add(10); // Duplikat diabaikan

    console.log(koleksiUnik.has(10)); // Output: true
    console.log(koleksiUnik.size);    // Output: 2 (bukan 3)

### d. WeakSet
Sama seperti relasi antara Map dan WeakMap, WeakSet adalah koleksi nilai unik yang hanya dapat menyimpan objek. Referensi objek di dalam WeakSet bersifat lemah dan tunduk pada pembersihan Garbage Collector.

## 3. Indexed Collections (Koleksi Berbasis Indeks)

Kategori ini mengelola sekuens data linear yang diurutkan dan diakses melalui indeks numerik (dimulai dari 0).

### a. Arrays
Array di JavaScript adalah struktur data list dinamis. Alokasi memori untuk array bersifat elastis (ukurannya dapat bertambah dan berkurang secara otomatis pada saat runtime), dan karena JavaScript adalah dynamically typed, satu array dapat menampung elemen dengan tipe data yang heterogen.

    const vektorData = ["Alpha", 42, true, { id: 1 }];

    // Mengakses berdasarkan indeks
    console.log(vektorData[0]); // Output: "Alpha"

    // Manipulasi data dinamis (Mutasi)
    vektorData.push("Omega"); // Menambah elemen ke akhir alokasi
    console.log(vektorData.length); // Output: 5

### b. Typed Arrays
Dalam ekosistem modern, JavaScript dituntut untuk berinteraksi dengan API tingkat rendah seperti manipulasi buffer audio/video, WebSockets, atau WebGL untuk grafis 3D. Array standar tidak efisien untuk tugas ini karena alokasinya memakan banyak overhead memori.

Di sinilah Typed Arrays hadir. Mereka menyediakan mekanisme fungsional untuk membaca dan menulis data biner mentah (raw binary data) di dalam memori internal (buffer). Typed Arrays memiliki panjang yang statis (fixed-length) dan elemen di dalamnya harus seragam dan berjenis numerik spesifik.


    // 1. Mengalokasikan blok memori mentah sebesar 16 byte (ArrayBuffer)
    const bufferMemori = new ArrayBuffer(16);

    // 2. Membuat sebuah "View" (Typed Array) untuk memanipulasi buffer tersebut
    // Menggunakan Int32Array (Integer 32-bit: setiap elemen butuh 4 byte)
    // Jadi, buffer 16 byte bisa menampung maksimal 4 elemen (16 / 4)
    const manipulasiBiner = new Int32Array(bufferMemori);

    manipulasiBiner[0] = 2048;
    manipulasiBiner[1] = 4096;

    console.log(manipulasiBiner[0]); // Output: 2048
