# All About variables

## 1. Deklarasi Variable
Dulu kita hanya punya var, tapi sejak standar ES6 (2015), muncul let dan const yang lebih aman.

- `var`: Cara lama. Bisa dideklarasikan ulang dan nilainya bisa diubah. (Sudah jarang dipakai karena sering bikin bingung).
- `let`: Cara modern. Nilainya bisa diubah, tapi tidak bisa dideklarasikan ulang di cakupan yang sama.
- `const`: Untuk nilai tetap (konstan). Sekali isi, tidak bisa diubah nilainya.

Contoh:

    var nama = "Purwa";
    var nama = "Andi"; // Boleh pakai var (tapi bahaya)

    let umur = 20;
    umur = 21; // Boleh diubah nilainya

    const PI = 3.14;
    // PI = 3.15; // Error! const tidak bisa diubah


## 2. Hoisting (Pengangkatan)
Hoisting adalah perilaku unik JavaScript. Secara harfiah artinya "diangkat".

Sebelum JavaScript menjalankan kode baris demi baris, dia punya kebiasaan membaca cepat (scanning) seluruh kodemu terlebih dahulu. Saat scanning, JavaScript akan mengangkat semua nama variabel ke bagian paling atas.

- Bagaimana jika pakai `var`?
Nama variablenya diangkat, tapi isinya ketinggalan.

        console.log(nama); 
        // Output: undefined (kotaknya ada, tapi dianggap belum ada isinya)

        var nama = "purwa";

- Bagaimana jika pakai `let` atau `const`?
Namanya tetap diangkat, tapi JavaScript melindunginya di area bernama Temporal Dead Zone. JavaScript melarangmu mengintip kotak itu sebelum baris pembuatannya dieksekusi.

NOTE: Temporal Dead Zone (TDZ) adalah `periode waktu di mana variabel yang dideklarasikan dengan let atau const sudah di-hoist (dibuat di memori) tetapi belum diinisialisasi.`

    console.log(kota); // ERROR! Cannot access 'kota' before initialization
    let kota = "Bandung";

## 3. Variable Naming Rules (Aturan Penamaan Variabel)
Bebas memberi nama variable, tapi ada hukum mutlak yang tidak boleh dilanggar:

1. Tidak boleh ada spasi. (Gunakan `namaLengkap`, bukan `nama lengkap`).
2. Hanya boleh `huruf`, `angka`, `underscore (_)`, dan `dolar ($)`. Karakter seperti @, #, - dilarang.
3. TIDAK BOLEH diawali angka. (orang1 itu boleh, tapi 1orang itu error).

Sensitif terhadap huruf besar/kecil (Case-sensitive). Variabel umur dan Umur dianggap sebagai dua variable yang berbeda.

Tips Profesional: Biasakan menggunakan format camelCase (huruf pertama kecil, kata selanjutnya diawali huruf besar). Contoh: namaPanggilan, totalHargaBarang.

## 4. Variable Scopes (Ruang Lingkup)
Dalam pemrograman, scope atau ruang lingkup variabel merupakan konsep yang menjelaskan di bagian mana suatu variabel dapat diakses, digunakan, atau dikenali oleh program. Scope berfungsi sebagai mekanisme pembatas akses agar variabel hanya dapat digunakan pada area tertentu sesuai kebutuhan, sehingga kode menjadi lebih terstruktur, aman, dan mudah dikelola.

Secara umum, JavaScript memiliki beberapa jenis scope, yaitu global scope, function scope, dan block scope.

1. Global Scope

Global scope adalah ruang lingkup untuk variabel yang dideklarasikan di luar fungsi maupun blok kode apa pun. Variabel dengan scope ini dapat diakses dari seluruh bagian program selama masih berada dalam konteks eksekusi yang sama.

    let bahasa = "JavaScript"; // Global Scope

    function tampilkanBahasa() {
    console.log(bahasa);
    }

    console.log(bahasa);

Pada contoh di atas, variabel bahasa dapat diakses baik di dalam fungsi maupun di luar fungsi karena berada pada ruang lingkup global.

2. Function Scope

Function scope merupakan ruang lingkup variabel yang hanya berlaku di dalam fungsi tempat variabel tersebut dideklarasikan. Variabel dengan scope ini tidak dapat diakses dari luar fungsi.

    function sapa() {
    let pesan = "Halo!"; // Function Scope
    console.log(pesan);
    }

    sapa();

    console.log(pesan); // ERROR

Pada contoh tersebut, variabel pesan hanya dikenali di dalam fungsi sapa(). Ketika dipanggil dari luar fungsi, program menghasilkan error karena variabel berada di luar jangkauan akses.

3. Block Scope

Block scope adalah ruang lingkup variabel yang dibatasi oleh blok kode yang diapit tanda kurung kurawal `{ }`, seperti pada `if`, `for`, atau `while`. Scope ini berlaku untuk variabel yang dideklarasikan menggunakan `let` dan `const`.

    if (true) {
    let rahasia = "12345"; // Block Scope
    console.log(rahasia);
    }

    console.log(rahasia); // ERROR

Variabel `rahasia` hanya dapat diakses di dalam blok `if`. Setelah eksekusi keluar dari blok tersebut, variabel tidak lagi dapat digunakan.

Perlu diperhatikan bahwa deklarasi menggunakan var tidak sepenuhnya mengikuti aturan block scope, karena var hanya memiliki function scope. Oleh sebab itu, penggunaan let dan const lebih disarankan dalam pengembangan JavaScript modern untuk menghindari kesalahan akses variabel.
